# 1. ビジョン

```
┌─────────────────────────────────────────────────────────────┐
│                                                              │
│   「自己肯定感が低い子供を、プログラミングで輝かせる」        │
│                                                              │
│   教育心理学に基づいたメソッドで                             │
│   子供の自己肯定感を高め、                                   │
│   その手段としてプログラミング教室を紹介する                 │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

## 1-1. 運営者の強み

| 経験・スキル | 内容 | 活用方法 |
|-------------|------|---------|
| デザイン学校卒業 | 専門教育を受けた | E-E-A-T（専門性の裏付け） |
| デザイナー実務5年 | 現場での実践経験 | プロフィールでの信頼性向上 |
| HP制作（コーディング） | HTML/CSS/JavaScript | プログラミング教育の文脈で語れる |
| 教育書籍を継続的に読む | 自己肯定感・非認知能力など | 記事の専門性・引用元として活用 |

### コーディングとプログラミングの関係

```
プログラミング（広義）
├── Webコーディング（HTML/CSS/JavaScript）← 運営者の経験
├── アプリ開発（Swift/Kotlin）
├── ゲーム開発（Unity/Scratch）
└── AI/データサイエンス（Python）
```

**→ Webコーディングはプログラミングの一種。運営者の経験はE-E-A-Tの専門性として活用。**

### 小学生向け市場の現実

調査の結果、小学生向けHTML/CSS・Webデザイン教室は**ほぼ存在しない**ことが判明。

| スクール | Webデザイン/HTML/CSS |
|---------|---------------------|
| Tech Kids School | なし（Scratch、Swift、C#が中心） |
| LITALICOワンダー | なし（ゲーム、ロボット、3DCGが中心） |
| デジタネ | あり（HTML&CSSコース、オンラインのみ） |

**小学生向けにHTML/CSSが少ない理由：**

| 理由 | 詳細 |
|------|------|
| 成果物が地味 | Scratchやマイクラはすぐ動く・遊べる。HTMLは静的で子供が飽きやすい |
| タイピングが壁 | 小学生はタイピングが遅い。コード入力が苦痛になる |
| エラーが分かりにくい | 閉じタグ忘れなど、小学生には特定しづらい |
| 親の期待と違う | 「プログラミング」と聞いてゲームやアプリを期待する |

**結論：小学生向けHTML/CSS教室の市場がない = 掲載する教室がない = 被リンク獲得できない**

### ターゲット教室（確定）

市場があり、被リンク獲得が可能なカテゴリに絞り込む。

| カテゴリ | 教室数 | 被リンク獲得 | 対応 |
|---------|--------|-------------|------|
| Scratch教室 | 多い | ◎ | **ターゲット** |
| マイクラ/ロブロックス教室 | 多い | ◎ | **ターゲット** |
| ロボット教室 | 多い | ◎ | **ターゲット** |
| HTML/CSS教室 | ほぼない | × | 対象外 |
| Webデザイン教室 | ほぼない | × | 対象外 |

```
【ターゲット教室】
・プログラミング教室（Scratch、マイクラ、ロブロックス等）
・ロボット教室

【対象外】
・Webデザイン教室（市場がない）
・HTML/CSS教室（市場がない）
```

### 運営者の経験の活用方法

デザイン/コーディング経験は「教室紹介」ではなく「E-E-A-Tの裏付け」として活用。

| 活用方法 | 詳細 |
|---------|------|
| プロフィール | 「デザイナー歴5年、プログラミング経験あり」で専門性を示す |
| 記事の視点 | 「クリエイティブな経験者」として教室を評価・紹介 |
| 独自の切り口 | 「デザイン思考 × 自己肯定感」など |

## 1-2. 書籍レビュー戦略

教育系書籍を読んでブログに書くことで、E-E-A-T（専門性・信頼性）を強化する。

### 読むべき書籍カテゴリ

| カテゴリ | 目的 | 例 |
|---------|------|-----|
| 自己肯定感 | メインテーマの深掘り | 「自己肯定感の教科書」中島輝 |
| 非認知能力 | サブテーマの強化 | 「学力の経済学」中室牧子 |
| 発達心理学 | 子供の成長理解 | 「子どもの発達心理学」 |
| 教育論 | 教育全般の知識 | 「MINDSET」キャロル・ドゥエック |
| プログラミング教育 | 専門領域の知識 | 「プログラミング教育はいらない」 |

### 書籍レビュー記事の書き方

```
タイトル: 【書評】「〇〇」を読んで学んだ子供の自己肯定感の育て方

構成:
1. この本を読んだ理由
2. 本の概要（3行でまとめ）
3. 印象に残った3つのポイント
4. 子育て・教育への活かし方
5. こんな人におすすめ
```

### 期待効果

| 効果 | 詳細 |
|------|------|
| E-E-A-T強化 | 「この人は本を読んで勉強している」という信頼性 |
| 記事の引用元 | 他の記事で「〇〇という本によると」と引用できる |
| 内部リンク | 書評記事 ↔ 関連記事 の相互リンク |
| コンテンツ量 | 月4冊 × 12ヶ月 = 48記事/年 |

**→ 月4冊ペースで読み、書評記事を書く**

## ビジネスモデル

```
教育心理学の記事で集客
    ↓
「自己肯定感を高めたい」保護者が集まる
    ↓
解決策としてプログラミング教室を提案
    ↓
提携教室へ送客
    ↓
収益化（アフィリエイト・送客手数料）
```

## 2-7. 地域ページ戦略

### コエテコとの戦い方

コエテコは198地域で1位を獲得しているが、正面から戦う。

| 方針 | 詳細 |
|------|------|
| 主要都市から着手 | 東京・大阪・名古屋・福岡など人口が多い地域 |
| 検索ボリュームがあるKWを狙う | 「地域名 + プログラミング教室」で勝負 |
| 無料掲載で教室数を増やす | 実際に教室を掲載 → 被リンク獲得 → ドメインパワー向上 |
| 記事の切り口で差別化 | 「自己肯定感を育てる教室」という視点 |

### キーワード戦略

```
◎ 「渋谷 プログラミング教室」
   → ボリュームがある。コエテコが強いが、無料掲載で被リンクを獲得しながら戦う

❌ 「渋谷 子供 自己肯定感 習い事」
   → 検索ボリュームが少なすぎる
```

**方針：**
- 検索ボリュームがある「地域名 + プログラミング教室」を狙う
- コエテコが強くても、無料掲載による被リンク獲得でドメインパワーを上げて勝負
- 1位を取れなくても、上位表示されれば収益化は可能
- 記事の切り口は「自己肯定感を育てる教室」で差別化

### ドメインパワー成長シナリオ

```
【初期】ドメインパワー 0
    ↓
無料掲載で教室を増やす（月20教室）
    ↓
教室からの被リンク獲得（獲得率20-30%）
    ↓
【6ヶ月後】被リンク40件 → ドメインパワー上昇
    ↓
地域KW「〇〇 プログラミング教室」で徐々に順位上昇
    ↓
【1年後】被リンク100件 → 主要地域で上位表示
    ↓
【2年後】被リンク200件 → コエテコと競合できるレベル
```

**ポイント：**
- 最初からコエテコに勝てなくても問題ない
- 被リンクを積み上げることでドメインパワーが徐々に上がる
- 2年かけて地域KWで上位表示を狙う長期戦略

## 2-4. コエテコの弱点

| 弱点 | 詳細 |

### コエテコが弱いキーワード（41件抽出）

| 条件 | 内容 |
|------|------|

| 条件2 | 難易度40以下 |
| 条件3 | 流入500以上 |

### 結論

```
コエテコが「弱い」キーワード41件のうち、
プログラミング関連はたった3件。

→ コエテコはプログラミング教育では非常に強い
→ SNS、画像検索、副業などは手薄（でもCVに遠い）
→ 「弱いところを奪う」より「別の土俵で戦う」が正解
```
