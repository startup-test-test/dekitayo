### なぜこの構造なのか？

#### 地域ページと教室ページを分ける理由

```
ユーザーの検索パターンは2種類ある：

パターン①：「場所」で探す
  「渋谷 プログラミング教室」
   → 渋谷にある教室を一覧で見たい
   → /ranking/prefecture/tokyo/city/shibuya/

パターン②：「教室名」で探す
  「テックキッズ 評判」
   → 特定の教室の詳細を見たい
   → /brands/tech-kids-school/
```

#### 分けることのメリット

```
❌ 分けない場合
/shibuya/tech-kids-school/
→ 「渋谷のテックキッズ」しか対応できない
→ 「テックキッズ全体」の検索に弱い

✅ 分けた場合（コエテコ方式）
/ranking/.../shibuya/     → 地域検索に対応
/brands/tech-kids-school/ → 教室名検索に対応
→ 両方のキーワードで上位を狙える
```

#### 内部リンクの相乗効果

```
教室ページ → 「渋谷校はこちら」→ 地域ページ
地域ページ → 「テックキッズスクール」→ 教室ページ

→ 相互リンクでどちらのページも強くなる
```

## できたよ！のサイト構造

### 基本方針

- **シンプル**：教室情報は地域ページに1箇所だけ（重複なし）
- 成長に合わせて拡張

### URL構造

```
dekitayo.jp/
│
├── /                        ← トップページ
│
│
│  ━━━ 記事 ━━━
│
├── /self-esteem/            ← 自己肯定感
│   └── /{記事名}/
│
├── /non-cognitive/          ← 非認知能力
│   └── /{記事名}/
│
├── /books/                  ← 書籍レビュー
│   └── /{書籍名}/
│
├── /column/                 ← コラム・体験談
│   └── /{記事名}/
│
├── /media/                  ← メディア・プレスリリース（転載）
│   └── /{記事名}/
│
│
│  ━━━ 教室 ━━━
│
├── /area/{都道府県}/{市区町村}/  ← 地域ページ（教室情報はここだけ）
│   例: /area/tokyo/shibuya/
│   内容：
│   ├── 教室一覧
│   ├── 各教室の基本情報
│   └── Googleレビュー（WordPress連携）
│
├── /listing/                ← 無料掲載案内
│
│
│  ━━━ 信頼性・法的 ━━━
│
├── /about/                  ← 運営者情報
├── /company/                ← 会社概要
├── /contact/                ← お問い合わせ
├── /privacy/                ← プライバシーポリシー
└── /terms/                  ← 利用規約
```

### ページ一覧

| ページ | URL | 役割 |
|--------|-----|------|
| トップ | `/` | サイトの顔 |
| 自己肯定感 | `/self-esteem/` | 検索流入（主力） |
| 非認知能力 | `/non-cognitive/` | 検索流入（主力） |
| 書籍レビュー | `/books/` | E-E-A-T強化 |
| コラム | `/column/` | 体験談など |
| メディア | `/media/` | プレスリリース転載 |
| 地域ページ | `/area/〇〇/〇〇/` | 教室紹介 + 被リンク獲得 |
| 無料掲載案内 | `/listing/` | 教室への案内 |
| 運営者情報 | `/about/` | 信頼性 |
| 会社概要 | `/company/` | 法的要件 |
| お問い合わせ | `/contact/` | 窓口 |
| プライバシーポリシー | `/privacy/` | 法的要件 |
| 利用規約 | `/terms/` | 法的要件 |

### 優先度

```
【Phase 1：最初に作る】
├── / （トップ）
├── /about/ （運営者情報）
├── /privacy/ （プライバシーポリシー）
├── /contact/ （お問い合わせ）
└── 最初の記事

【Phase 2：記事を増やす】
├── /self-esteem/
├── /non-cognitive/
├── /books/
└── /column/

【Phase 3：教室掲載を始める】
├── /listing/ （無料掲載案内）
├── /area/〇〇/〇〇/ （地域ページ）
├── /company/ （会社概要）
└── /terms/ （利用規約）
```

### なぜこの設計か

```
✅ 教室情報は地域ページだけ
   → 重複コンテンツなし
   → SEO的に安全
   → 管理が楽

✅ Googleレビュー連携
   → 自分で口コミを集める必要なし
   → すぐに信頼性のあるページが作れる

✅ 記事が主力
   → コエテコと同じ土俵で戦わない
   → 「自己肯定感」「非認知能力」で差別化
```

### コエテコとの違い
