自己理解プログラム

最終的なゴール

誰もが自分の強みだけで生きていける「自立の仕組み」を完成させる

ビジョン

「一人ひとりの『ものさし』で、生きられる世界をつくる」

学歴や偏差値という「社会のものさし」ではなく、自分の強みや個性という「自分のものさし」で評価される世界。誰もが自分らしく生きられる社会。

仕事の目的

諦めている状態の対象を → 自分らしい戦略を見つけて、自由に選べる状態にする

Before（諦めている状態）After（自由に選べる状態）
自分には無理だと思っている自分の強みを活かせる
選択肢がないと感じている選択肢が増える
自己肯定感が低い自信を持って決断できる

価値観 TOP5

順位価値観キーワード
1位自由自分で選択できる、束縛されない
2位希望未来に期待できる、可能性がある
3位夢中没頭できる、時間を忘れる
4位好奇心新しいことを知りたい、学びたい
5位安心不安がない、安定している

ストレングスファインダー TOP14

順位資質説明
1最上志向平均ではなく卓越を求める
2分析思考データや事実に基づいて判断する
3適応性変化に柔軟に対応する
4着想新しいアイデアを生み出す
5戦略性複数の選択肢から最適解を見つける
6自我自分の存在価値を認められたい
7未来志向未来のビジョンを描く
8目標志向明確なゴールに向かって進む
9内省自分自身について深く考える
10原点思考過去から学び、本質を見抜く
11親密性深い人間関係を築く
12自己確信自分の判断を信じる
13社交性新しい人と出会うことを楽しむ
14収集心情報を集めることが好き

長所パターン TOP5

  1. 戦略やアイデアを練るために、1人で深く思考する時間や環境を作る
  2. 失敗しないように、成功事例を調査して分析し、真似する
  3. 目的を達成するために、計画を明確にする
  4. 複雑な情報を分析して、相手が判断しやすい情報にする
  5. 未来の理想像から逆算し、具体的な数値目標やタスクを設定する

本当にやりたいこと

教育心理学の成功事例を調べてユニークなアイデアを考えて提案する

  • 得意なこと（才能）：成功事例を調査・分析して、戦略を立てる
  • 好きなこと（興味）：教育心理学、自己肯定感、非認知能力
  • 大事なこと（価値観）：自由、希望、夢中

ロゴ

サイト名

できたよ!

「！」があることで喜び・達成感が伝わる。子供が嬉しそうに報告している感じが出る。

イメージ案

  • 「！」を星や光で表現
  • 手を上げて喜ぶシルエット
  • チェックマーク + 笑顔
  • 上向きの要素（成長・達成）

トーン＆マナー

温かみがあるけど、信頼できる

親しみやすいけど、専門的

明るいけど、落ち着いている

→ 「子供向け」ではなく「親向け」のデザイン

禁止事項

  • 子どもっぽすぎるデザイン：信頼性が下がる
  • 暗い・重い印象：コンセプトと矛盾
  • 複雑すぎるレイアウト：情報が伝わらない
  • 原色の多用：安っぽくなる
  • 過度な装飾：読みにくくなる

Next.jsを選んだ理由

理由説明
SEOに強いサーバーサイドレンダリング対応
情報が多い困った時に調べやすい
Vercelと相性◎デプロイが簡単、無料枠あり
WordPress連携実績多いヘッドレスCMSの定番構成

Phase 1：インフラ準備

ドメイン設定

手順作業内容備考
1ドメイン取得dekitayo.jp（ムームードメイン）
2ネームサーバー変更Xserverに向ける
3サブドメイン作成cms.dekitayo.jp
4DNS反映確認1〜2時間待つ

Phase 2：WordPress構築

WordPressインストール

  1. 「WordPress簡単インストール」をクリック
  2. 対象ドメイン：cms.dekitayo.jp を選択
  3. サイトURL、ブログ名、ユーザー名、パスワードを入力
  4. 「インストール」をクリック

WordPress初期設定

設定項目操作
設定 → 表示設定「検索エンジンがサイトをインデックスしないようにする」に✅
設定 → パーマリンク「投稿名」を選択
プラグインHello Dolly を削除

Phase 3：プラグインインストール

プラグイン用途インストール方法
WPGraphQLGraphQL API提供プラグイン検索
Advanced Custom Fieldsカスタムフィールドプラグイン検索
WPGraphQL for ACFACFをGraphQL対応GitHub からダウンロード

Phase 6：Next.js連携

プロジェクト作成

npx create-next-app@latest dekitayo-frontend --typescript --tailwind --app
cd dekitayo-frontend

環境変数設定

WORDPRESS_API_URL=https://cms.dekitayo.jp/graphql
WORDPRESS_AUTH_USER=Basic認証ユーザー名
WORDPRESS_AUTH_PASS=Basic認証パスワード

GraphQLクライアント設定

npm install graphql-request graphql

プレスリリース手順

掲載サービス

サービス料金特徴
PR TIMES3万円〜最大手、メディア転載多い
@Press3万円〜配信先が多い
valuepress無料〜無料プランあり
PRTIMES STORY無料ストーリー形式

手順

Step 1：プレスリリース作成

  1. タイトル作成（30文字以内）
  2. リード文作成（概要）
  3. 本文作成
  4. 画像準備（ロゴ、サービス画像）
  5. 会社概要作成

Step 2：配信サービスに登録

  1. アカウント作成
  2. 企業情報登録
  3. プレスリリース入稿
  4. 画像アップロード
  5. プレビュー確認

Step 3：配信

  1. 配信日時設定
  2. 配信実行
  3. 掲載確認

Step 4：自社サイトに転載

  1. /media/ ページに記事作成
  2. プレスリリース内容を転載
  3. 配信元へのリンク設置
  4. 公開

プレスリリースのタイミング

タイミング内容
サイト公開時サービス開始のお知らせ
掲載教室数達成時「掲載100教室突破」など
新機能追加時機能アップデート
調査レポート発表時独自調査の発表
